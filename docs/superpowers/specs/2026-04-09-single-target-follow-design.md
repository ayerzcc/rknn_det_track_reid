# 单目标跟随优化设计

## 目标

面向机器狗人员跟随场景，将当前“通用多人跟踪”策略收敛为“稳定锁定一个目标并尽量保住旧 ID”的单目标跟随策略。核心目标不是提升通用 MOT 指标，而是减少目标丢失、跳 ID 和控制抖动。

## 当前问题

当前系统虽然具备检测、跟踪和 ReID 恢复能力，但整体仍按多目标逻辑组织。对于机器狗跟随场景，这带来三个直接问题：

1. 目标切换过于自由，系统容易从跟随对象跳到其他人。
2. 目标暂时出画、遮挡或被别的 active track 抢占 detection 后，旧 ID 恢复优先级不够高。
3. 即使视觉轨迹连续，控制输入若直接使用 bbox 中心和面积，也会导致机器人运动抖动。

## 推荐架构

推荐引入“单目标锁定器 + 目标专属恢复 + 控制平滑”三层结构，覆盖现有 `BoTSORT` 路径。

### 1. 单目标锁定器

系统始终维护一个 `target_track_id` 和 `target_state`。

- `UNLOCKED`：未锁定目标，系统根据候选评分选择一个跟随目标。
- `LOCKED_VISIBLE`：目标当前可见，持续跟踪并更新目标状态。
- `LOCKED_LOST_SHORT`：目标短时丢失，优先进行旧 ID 恢复，不允许轻易切换到新的人。
- `LOCKED_LOST_LONG`：目标长时间不可恢复，才允许解锁并重新选择。

### 2. 目标专属恢复

对当前目标单独开高优先级恢复路径。

- 目标进入 `LOST_SHORT` 后，优先使用更长的恢复窗口和更长的 gallery 历史。
- 新进入画面的 detection 不仅和 active tracks 做关联，也要和当前目标的 gallery 做 ReID 比较。
- 对目标专属恢复，可比普通 lost track 更激进，但只针对当前目标，不影响其他轨迹。

### 3. 控制平滑层

对送给机器狗控制器的视觉观测做稳定化。

建议平滑以下量：

- 目标框中心 `cx`
- 目标框高度/面积
- 推导出的目标距离

避免机器人左右摆头、前后抖动。

## 目标选择策略

首次锁定目标时，不应只看最近的检测框，而应综合评分：

- 距离画面中心的距离
- bbox 面积/高度
- 检测置信度
- 与查询目标或历史 gallery 的相似度

推荐统一成一个目标评分函数：

`target_score = center_score + size_score + det_score + reid_score`

选择分数最高者作为首个跟随目标。

## 目标切换策略

锁定目标后，默认禁止切换。

只有以下情况才允许切换：

- 目标进入 `LOST_LONG`
- 当前候选与历史目标相似度明显更高，且连续多帧稳定

这意味着：

- 不因为一两帧检测抖动就切目标
- 不因为旁边有人出现就切目标

## 参数建议

针对当前场景，参数建议独立于多目标默认值：

- `target_recovery_window`: 20 ~ 60
- `target_gallery_history`: 20 ~ 50
- `target_similarity_threshold`: 略低于普通恢复阈值，但仍需结合几何约束
- `target_switch_patience`: 至少连续 5 ~ 10 帧确认后再切

第一阶段建议先做参数化，不直接写死默认值。

## 与现有系统边界

第一阶段不建议重写 `BoTSORT` 内核，而是优先在外层增加“单目标锁定器”。

这样有三点好处：

1. 风险小，不破坏现有多目标基础行为。
2. 便于和已有 `tracker_test`、`ReIDRecovery` 模块复用。
3. 便于 A/B 测试和逐步回退。

## 第一阶段实现范围

仅围绕 `BoTSORT` 路径做以下内容：

1. 新增单目标状态机
2. 新增目标选择评分
3. 新增目标专属恢复参数
4. 新增控制平滑输出

暂不做：

- 多目标统一竞争重分配
- 复杂多目标重排序
- 训练新 ReID 模型

## 验证标准

第一阶段验证只看单目标跟随相关指标：

- 目标出画后再回来，旧 ID 恢复次数是否增加
- 目标跟随连续性是否提升
- 机器人控制量是否更平滑
- 误切换到其他人次数是否下降

## 推荐下一步

下一步进入实现计划时，建议只围绕 `BoTSORT` 做最小闭环：

1. 单目标锁定器
2. 目标选择评分
3. 目标专属恢复窗口/历史参数
4. 控制量平滑

验证通过后，再决定是否把同样策略扩展到 `OA-BoTSORT`。
